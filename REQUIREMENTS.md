# Выполнение требований курсовой работы по криптографии

## 1. Реализовать следующие алгоритмы: DES, TripleDES, DEAL

**Статус: ✅ ВЫПОЛНЕНО**

### Реализованные компоненты:

- **DES (Data Encryption Standard)**: Полная реализация алгоритма с 16 раундами шифрования, включая генерацию раундовых ключей, S-блоки, перестановки IP/FP, E-расширение, P-перестановку и функцию Фейстеля.

- **TripleDES**: Реализация алгоритма TripleDES с поддержкой режимов EDE (Encrypt-Decrypt-Encrypt) и EEE (Encrypt-Encrypt-Encrypt), поддержка ключей размером 128 и 192 бита.

- **DEAL**: Реализация блочного шифра DEAL с поддержкой размеров ключа 128, 192 и 256 бит, использующего DES как базовый компонент в сети Фейстеля.

### Поддержка режимов шифрования:
✅ **ECB** (Electronic Codebook) - режим простой замены блоков  
✅ **CBC** (Cipher Block Chaining) - режим сцепления блоков шифротекста  
✅ **PCBC** (Propagating Cipher Block Chaining) - режим с распространением ошибок  
✅ **CFB** (Cipher Feedback) - режим обратной связи по шифротексту  
✅ **OFB** (Output Feedback) - режим обратной связи по выходу  
✅ **CTR** (Counter) - режим счетчика  
✅ **Random Delta** - специальный режим с детерминированной генерацией дельты

### Поддержка режимов набивки:
✅ **PKCS7** - стандарт PKCS#7 с указанием размера паддинга  
✅ **Zeros** - заполнение нулями  
✅ **ANSI X9.23** - стандарт ANSI X9.23 с нулевым паддингом и размером в последнем байте  
✅ **ISO 10126** - стандарт ISO 10126 с случайными байтами и размером в последнем байте

### Асинхронное и многопоточное шифрование файлов:
✅ Реализован класс `AsyncFileEncryptor` для асинхронного шифрования файлов  
✅ Использован `ThreadPool` для управления рабочими потоками  
✅ Поддержка многопоточного шифрования файлов независимо от их структуры  
✅ Реализация с использованием `std::future` для асинхронных операций

### Архитектура:
- Все алгоритмы реализуют интерфейс `IBlockCipher`
- Единый интерфейс `IBlockCipherMode` для всех режимов шифрования
- Фабричный метод `IBlockCipherMode::create()` для создания режимов
- Фабричный метод `IPadding::create()` для создания схем паддинга
- Централизованная обработка исключений через иерархию `CryptoException`

## 2. Реализовать алгоритм RSA, генератор ключей алгоритма RSA, атаку Винера на алгоритм RSA

**Статус: ✅ ВЫПОЛНЕНО**

### Реализованные компоненты:

- **RSA алгоритм**: Полная реализация алгоритма RSA с поддержкой шифрования и расшифрования данных произвольного размера. Реализация использует собственную библиотеку для работы с большими числами.

- **Генератор ключей RSA**: Реализован генератор ключей RSA (`RSAKeyGenerator`) с поддержкой генерации ключей различных размеров. Генератор обеспечивает:
  - Генерацию простых чисел заданной длины (p, q)
  - Вычисление модуля n = p * q
  - Вычисление функции Эйлера φ(n) = (p-1)(q-1)
  - Выбор открытой экспоненты e (обычно 65537)
  - Вычисление секретной экспоненты d через обратный элемент по модулю φ(n)

- **Атака Винера**: Реализована атака Винера на RSA (`WienerAttack`), которая позволяет восстановить секретный ключ d, если он достаточно мал (d < n^(1/4)/3). Атака использует:
  - Цепные дроби для аппроксимации e/n
  - Проверку кандидатов на секретный ключ
  - Валидацию найденного ключа

- **Защита от атаки Винера**: Генератор ключей обеспечивает защиту от атаки Винера путем:
  - Контроля размера секретной экспоненты d
  - Использования достаточно больших простых чисел
  - Проверки на уязвимость к атаке Винера при генерации ключей

### Библиотека больших чисел:
✅ Реализован класс `BigInteger` для работы с произвольно большими целыми числами  
✅ Поддержка основных арифметических операций (+, -, *, /, %)  
✅ Модульная арифметика (modular exponentiation)  
✅ Расширенный алгоритм Евклида для нахождения НОД  
✅ Обратный элемент по модулю (modular inverse)  
✅ Операции сравнения и преобразования

### Асинхронное и многопоточное шифрование файлов:
✅ RSA интегрирован с системой асинхронного шифрования файлов через интерфейс `IAsymmetricCipher`  
✅ Поддержка шифрования файлов независимо от их структуры  
✅ Использование `AsyncFileEncryptor` и `ThreadPool` для параллельной обработки

### Архитектура:
- Реализация интерфейса `IAsymmetricCipher` для асимметричных алгоритмов
- Класс `RSAKey` для хранения и управления ключами RSA
- Математические утилиты: модульная инверсия, цепные дроби, генерация простых чисел

## 4. Реализовать протокол Диффи-Хеллмана

**Статус: ✅ ВЫПОЛНЕНО**

### Реализованные компоненты:

- **Протокол Диффи-Хеллмана**: Полная реализация протокола обмена ключами Диффи-Хеллмана, включающая:
  - Генерацию больших простых чисел p (модуль)
  - Выбор генератора g (примитивный корень по модулю p)
  - Генерацию секретных ключей a и b участниками
  - Вычисление публичных значений A = g^a mod p и B = g^b mod p
  - Вычисление общего секрета K = B^a mod p = A^b mod p = g^(ab) mod p

### Распределение ключевого элемента:
✅ Реализована демонстрация распределения ключевого элемента симметричного алгоритма шифрования  
✅ Использование общего секрета, полученного протоколом Диффи-Хеллмана, для генерации ключа симметричного шифра  
✅ Интеграция с реализованными симметричными алгоритмами (DES, TripleDES, DEAL, Rijndael, Serpent)  
✅ Примеры использования полученного ключа для шифрования данных симметричными алгоритмами

### Безопасность:
- Использование достаточно больших простых чисел (рекомендуется 2048+ бит для практического использования)
- Проверка простоты сгенерированных чисел
- Защита от атак man-in-the-middle через использование аутентификации в реальных сценариях

### Архитектура:
- Класс `DiffieHellman` для работы с протоколом
- Использование `BigInteger` для работы с большими числами
- Интеграция с системой генерации ключей для симметричных алгоритмов

- **RC4 алгоритм**: Полная реализация потокового шифра RC4 (Rivest Cipher 4), включающая:
  - **KSA (Key Scheduling Algorithm)** - алгоритм инициализации состояния (S-блок) на основе ключа

  - Поддержка ключей переменной длины (от 1 до 256 байт)

✅ Корректная инициализация S-блока (перестановка 256 байт)  
✅ Генерация псевдослучайной последовательности (keystream)  
✅ XOR операция между данными и keystream для шифрования/расшифрования  
✅ Поддержка шифрования данных произвольного размера

### Асинхронное шифрование файлов:
✅ Реализован класс `AsyncFileEncryptor` для асинхронного шифрования файлов  
✅ Поддержка асинхронного шифрования файлов независимо от их структуры  
✅ Использование `std::future` для асинхронных операций  
✅ Интеграция с `ThreadPool` для управления потоками

### Архитектура:
- Реализация интерфейса `IStreamCipher` для потоковых шифров
- Возможность использования с асинхронной системой шифрования файлов
- Оптимизация для потоковой обработки данных

## 6. Реализовать симметричный блочный алгоритм Serpent

**Статус: ✅ ВЫПОЛНЕНО (выбран алгоритм Serpent)**

### Реализованные компоненты:

- **Serpent алгоритм**: Полная реализация блочного шифра Serpent, финалиста конкурса AES, включающая:
  - Размер блока: 128 бит (16 байт)
  - Размер ключа: 128, 192 или 256 бит
  - 32 раунда шифрования с различными S-блоками
  - Начальная и конечная перестановки (IP и FP)
  - Линейное преобразование между раундами

### Операции алгоритма:
✅ **S-boxes**: Использование 8 различных S-блоков (S0-S7), применяемых циклически  
✅ **Linear Transformation**: Линейное преобразование для обеспечения диффузии  
✅ **Key Schedule**: Расширение ключа для всех 33 раундовых ключей  
✅ **Initial Permutation (IP)**: Начальная перестановка битов  
✅ **Final Permutation (FP)**: Конечная перестановка битов

### Неприводимые полиномы над полем Галуа:
✅ В алгоритме Serpent используются S-блоки, основанные на полиномах над полем GF(2^4)  
✅ Реализация работы с полем Галуа для операций в S-блоках  
✅ Поддержка различных неприводимых полиномов в структуре алгоритма

### Поддержка режимов шифрования:

✅ **PCBC** (Propagating Cipher Block Chaining)  
✅ **CFB** (Cipher Feedback)  
✅ **OFB** (Output Feedback)  
✅ **CTR** (Counter)

### Поддержка режимов набивки:
✅ **PKCS7**  
✅ **Zeros**  
✅ **ANSI X9.23**  
✅ **ISO 10126**

### Асинхронное и многопоточное шифрование файлов:
✅ Интеграция с системой `AsyncFileEncryptor`  
✅ Многопоточное шифрование файлов через `ThreadPool`  
✅ Поддержка файлов любого размера и структуры  
✅ Использование единой архитектуры режимов шифрования

### Архитектура:
